# Архитектура приложения Clicker 2.0

## Общая архитектура

Clicker 2.0 реализует архитектуру с множественными агентами, где каждый бот работает как независимый виртуальный браузер с собственной логикой FSM (конечного автомата). Система состоит из следующих ключевых компонентов:

### 1. Главный контроллер ([`main.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/main.py))

Основной точкой входа в приложение является `main.py`, который:

- Загружает конфигурацию из `config.yaml`
- Создает анализатор GPU для обработки изображений
- Инициализирует несколько виртуальных ботов (до 12)
- Для каждого бота создает соответствующую логику FSM
- Запускает основной цикл, в котором:
  - Захватывает кадры от каждого бота
  - Выполняет шаги FSM с этими кадрами
  - Управляет синхронизацией между ботами

### 2. Виртуальная среда бота ([`scripts/env_bot.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/env_bot.py))

Каждый бот представляет собой изолированную виртуальную среду, которая:

- Использует Xvfb (X Virtual Framebuffer) для создания виртуального дисплея
- Запускает легковесный оконный менеджер Fluxbox
- Предоставляет доступ VNC для удаленного просмотра
- Запускает оптимизированный браузер Chromium с различными флагами производительности
- Управляет профилем пользователя для сохранения сессии

#### Оптимизации браузера:
- Полностью отключен JavaScript
- Отключено аппаратное ускорение
- Отключены изображения, шрифты и анимации
- Ограничение частоты кадров до 1 FPS
- Отключены уведомления и фоновые сетевые соединения

### 3. Логика FSM ([`scripts/bot_logic.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/bot_logic.py))

Каждый бот связан с FSM (конечным автоматом), который управляет его поведением:

- Имеет начальное состояние, определенное в конфигурации
- Состояния определяют, что искать на экране и какие действия выполнять
- Состояния могут иметь таймауты для сброса в случае застревания
- Переходы между состояниями основаны на условиях (наличие шаблона, валидность JSON и т.д.)

#### Основные методы FSM:
- `execute_step()`: Выполняет один шаг FSM
- `_run_action()`: Выполняет конкретное действие (клик, вставка текста и т.д.)
- `get_clipboard()`: Получает содержимое буфера обмена
- `is_json_valid()`: Проверяет валидность JSON

### 4. Анализатор GPU ([`scripts/gpu_analyzer.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/gpu_analyzer.py))

Компонент для обработки изображений, который:

- Использует OpenCV для сопоставления шаблонов
- Кэширует загруженные шаблоны для повышения производительности
- Находит лучшие совпадения между текущим кадром и шаблонами
- Возвращает координаты найденных элементов

## Конфигурация

### Структура config.yaml

Файл конфигурации определяет:

- **global**: Глобальные параметры (размер экрана, FPS, путь к данным)
- **prompts**: Подсказки для разных моделей ИИ
- **bots**: Конфигурации отдельных ботов (проект, сайт, модель, область интереса)
- **sites**: Сценарии взаимодействия для каждого сайта

Каждый сайт имеет свои сценарии, которые определяют:

- Последовательность состояний FSM
- Шаблоны для распознавания элементов
- Действия для выполнения
- Условия перехода между состояниями

## Поток выполнения

1. **Инициализация**: Запуск всех виртуальных ботов и создание FSM
2. **Цикл обработки**:
   - Каждый бот захватывает текущий кадр
   - FSM анализирует кадр, используя анализатор GPU
   - Если найдено ожидаемое состояние, выполняется соответствующее действие
   - Проверяется условие перехода к следующему состоянию
   - Обновляется текущее состояние FSM
3. **Мониторинг**: Система продолжает работу до получения сигнала остановки

## Безопасность и стабильность

- Изолированные профили браузеров предотвращают перекрестные помехи
- Механизмы очистки кэша удаляют только ненужные данные, сохраняя сессии
- Таймауты предотвращают застревание в одном состоянии
- Проверка валидности JSON обеспечивает целостность данных

Эта архитектура позволяет эффективно масштабировать систему для одновременной работы с несколькими ИИ-сервисами, минимизируя использование ресурсов и обеспечивая надежную автоматизацию.
