# Clicker 2.0 - Автоматизированная система взаимодействия с ИИ

## Обзор проекта

Clicker 2.0 - это автоматизированная система, которая управляет несколькими виртуальными браузерами для взаимодействия с различными сервисами искусственного интеллекта, такими как Perplexity, ChatGPT, Gemini и Qwen. Приложение использует компьютерное зрение и сопоставление шаблонов для идентификации элементов пользовательского интерфейса и автоматизации взаимодействий.

## Архитектура приложения

### Основные компоненты

1. **Контроллер главного процесса ([`main.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/main.py))**
   - Инициализирует несколько виртуальных сред браузера (до 12 ботов)
   - Создает анализатор GPU для обработки изображений
   - Настраивает контроллеры конечных автоматов (FSM) для каждого бота
   - Запускает непрерывный цикл, который захватывает кадры и выполняет шаги FSM

2. **Виртуальная среда браузера ([`scripts/env_bot.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/env_bot.py))**
   - Создает изолированные виртуальные дисплеи с использованием Xvfb
   - Запускает браузеры Chromium с экстремальными настройками оптимизации ресурсов
   - Реализует доступ VNC для мониторинга
   - Управляет профилями браузера с сохранением сессии
   - Предоставляет возможности захвата кадров для компьютерного зрения

3. **Логика конечного автомата ([`scripts/bot_logic.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/bot_logic.py))**
   - Реализует логику автоматизации на основе состояний
   - Использует сопоставление шаблонов для обнаружения элементов интерфейса
   - Выполняет действия: клики, клавиатурные вводы и операции с буфером обмена
   - Обрабатывает переходы между состояниями на основе условий
   - Включает управление таймаутами для сброса состояний

4. **Анализатор GPU ([`scripts/gpu_analyzer.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/gpu_analyzer.py))**
   - Выполняет сопоставление шаблонов с использованием OpenCV
   - Оптимизирован для обработки на GPU с использованием UMat
   - Кэширует шаблоны для повышения производительности
   - Определяет координаты соответствующих элементов интерфейса

5. **Модуль проверки JSON ([`scripts/verification_saved_json.py`](file:///home/dmitrii/Documents/projects/clicker_2.0/scripts/verification_saved_json.py))**
   - Проверяет ответы JSON от сервисов ИИ
   - Очищает и восстанавливает неправильно сформированный JSON
   - Нормализует форматы данных (статей, актов, номеров)
   - Проверяет действительность сохраненных ответов JSON

## Структура конфигурации

Приложение настраивается через YAML-файлы:
- **config.yaml**: Содержит глобальные настройки, подсказки, конфигурации ботов и сценарии для конкретных сайтов
- **bots_home.yaml**: Определяет индивидуальные конфигурации ботов с ROI (область интереса) и назначениями задач

## Ключевые особенности

1. **Архитектура с несколькими ботами**: Поддерживает до 12 одновременных виртуальных браузеров
2. **Распознавание на основе шаблонов**: Использует шаблоны изображений для определения элементов интерфейса
3. **Автоматизация с конечным автоматом**: Сложная обработка сценариев через FSM
4. **Оптимизация ресурсов**: Экстремальная оптимизация браузера для снижения потребления ресурсов
5. **Интеграция компьютерного зрения**: Обработка изображений в реальном времени для взаимодействия с интерфейсом
6. **Операции с буфером обмена**: Функциональность копирования/вставки для передачи данных
7. **Обработка JSON**: Автоматическая проверка и очистка ответов ИИ

## Поддерживаемые платформы и модели

Приложение поддерживает несколько платформ ИИ:
- Perplexity (с моделями Grok, Claude Sonnet, Gemini)
- ChatGPT (с различными режимами)
- Qwen (режим веб-поиска)
- Gemini

Каждая платформа имеет специфические сценарии автоматизации, определенные в конфигурационном файле с настраиваемыми шаблонами и потоками взаимодействия.

## Назначение системы

Приложение предназначено для автоматизации сбора данных из различных сервисов ИИ путем имитации человеческого взаимодействия с веб-интерфейсами, захвата ответов и их последующей обработки в структурированный формат JSON для дальнейшего анализа. Система особенно полезна для генерации гипотез и правовых норм, как видно из ссылок на "hypos_norm" (нормализация гипотез) по всему коду.
