# VNC Monitor в main.py - Автоматический мониторинг и рестарт x11vnc

## Что это?

Встроенный в `main.py` монитор VNC серверов который:
- ✓ Проверяет все VNC серверы каждые 30 секунд
- ✓ **Автоматически перезапускает x11vnc** если он упал
- ✓ **НЕ перезапускает бота** — только VNC сервер
- ✓ Работает пока запущен `main.py`

## Почему только x11vnc?

**x11vnc — это просто "транслятор" экрана:**
- Не хранит данные браузера
- Не влияет на состояние бота
- При рестарте сохраняется вся сессия браузера

**Бот продолжает работать:**
- Браузер открыт
- Профиль сохранён
- FSM выполняет логику
- VNC клиент переподключается автоматически

## Как работает?

```
main.py запущен
  ↓
VNC Monitor активирован
  ↓
Каждые 30 секунд:
  ├─ Проверка Xvfb (:100-:109)
  ├─ Проверка VNC портов (5900-5909)
  ├─ Проверка x11vnc процессов
  │
  ├─ Если VNC упал → рестарт x11vnc
  │  └─ Успех → бот работает дальше ✓
  │  └─ Не успех → лог в консоль
  │
  └─ Если Xvfb упал → лог (требуется ручное вмешательство)
```


### Если не удалось восстановить:
```

[!] VNC Monitor: Не удалось восстановить VNC для бота 1
[!] VNC Monitor: Попробуйте перезапустить бота вручную
```

### Статус (каждые 5 минут):
```
[*] VNC Monitor: Все 4 ботов OK (проверка #10)
```

## Настройки

В `main.py` строка 39:
```python
vnc_monitor = VNCHealthMonitor(bot_count=bots_count, check_interval=30.0)
```

Параметры:
- `bot_count` - количество ботов (берётся из config_main.yaml)
- `check_interval` - интервал проверки в секундах (по умолчанию 30)

## Отличие от cron watchdog

| VNC Monitor в main.py | Cron Watchdog |
|----------------------|---------------|
| Работает пока main.py запущен | Работает всегда через cron |
| Может рестартануть всего бота | Только x11vnc |
| Интегрирован в логи бота | Отдельные логи |
| **Рекомендуется** | Не нужен |

## Использование

Просто запустите `main.py`:
```bash
python3 main.py
```

Монитор автоматически:
1. Запустится вместе с ботами
2. Будет проверять VNC каждые 30 секунд
3. При проблемах попытается восстановить
4. Если не вышло - перезапустит бота

```bash
python3 main.py 2>&1 | grep -i vnc
```

```bash
for port in 5900 5901 5902 5903; do
  echo -n "$port: "
  ss -tlnp | grep -q ":$port" && echo "✓" || echo "✗"

```

## Решение проблем

### Монитор не запускается:
Проверьте что импорты в main.py правильные:
```python
from scripts.vnc_monitor import VNCHealthMonitor
```

### Callback не срабатывает:
Проверьте что callback зарегистрирован для каждого бота:
```python
vnc_monitor.register_restart_callback(i, on_vnc_restart_needed)
```

### Бот не рестартует:
Проверьте что `vnc_problem_bots` обрабатывается в главном цикле (Шаг 0.5).
